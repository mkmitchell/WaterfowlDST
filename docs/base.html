Module Waterfowl

Defines Waterfowlmodel class which is used for storing parameters and doing calculations for the waterfowl decision support tool.

class waterfowlmodel.base.Waterfowlmodel(aoi, aoiname, wetland, kcalTable, crosswalk, demand, urban, binIt, binUnique, extra, fieldtable, scratch, classAttr)

Stores waterfowl model parameters and methods.

Creates a waterfowl model object.

Parameters

  • aoi (str) – feature dataset of area of interest

  • aoiname (str) – Name of area of interest used for unique naming

  • wetland (str) – Wetland feature dataset (e.g. National Wetland Inventory)

  • kcalTable (str) – CSV file containing two columns [habitat type, kilocalorie value by hectares]

  • crosswalk (str) – CSV file relating wetland habitat types to kcal csv table

  • demand (str) – Feature dataset of NAWCA stepdown Duck Use Days objectives

  • urban (str) – Raster dataset of impervious and built features

  • binIt (str) – Feature dataset of aggregation features (e.g. HUC12 watersheds)

  • binUnique (str) – Unique fields for aggregation (e.g. HUC12 ID number)

  • extra (str) – List of extra feature datasets and their corresponding crossover table

:param fieldtable Crosswalk table to standardize field names and aliases :type fieldtable str :param scratch: Scratch geodatabase location :type scratch: str

aggproportion(aggTo, aggData, IDField, aggFields, dissolveFields, scratch, cat, aggStat='SUM')

Calculates proportional sum aggregate based on area within specified columns of a given dataset to features from another dataset. Example: One aggData feature overlays two aggTo features (A abd B). The one aggData feature has 100 kcal. 60% of the aggData feature is in aggTo feature A and 40% in aggTo feature B. This method will assign 6 0kcal to feature A and 40 kcal to feature B.

Parameters

  • aggTo (str) – Spatial dataset used as the aggregation feature. Data will be binned to the features within this dataset.

  • aggData (str) – Spatial data to be aggregated

  • IDField (str) – ID field used for data aggregation

  • aggFields (str) – Field for aggregation

  • dissolveFields (str) – Field for dissolving bins after union

  • scratch (str) – Scratch geodatabase location

  • cat (str) – Spatial dataset used as the aggregation feature. Data will be binned to the features within this dataset.

  • aggStat (str) – Aggregation statistic. Default is SUM

Returns

Spatial Sum aggregate of aggData within supplied bins

Return type

str

dstOutput(mergebin, outputgdb)

Runs energy difference between NAWCA stepdown objectives and available habitat.

Calculations: Energy supply

Total habitat energy within huc - THabNrg Total habitat hectares within huc - THabHA

Energy demand

LTA and X80 DUD by huc - TLTADUD anc X80DUD LTA and X80 Demand by huc - TLTADemand and X80Demand LTA and X80 Population objective by huc - LTAPopObj and X80PopObj

Protected lands

Total protected hectares by huc - ProtHA

Protected habitat hectares and energy

Total protected hectares - ProtHabHA Total protected energy - ProtHabNrg

Weighted mean and calculations based off of it

Weighted mean kcal/ha with weight being Total habitat energy Energy Protection needed - NrgProtRq Restoration HA based off of weighted mean - RstorHA Protection HA based off weighted mean - RstorProtHA

Parameters

  • mergebin (list) – List that holds all datasets to be merged for output

  • outputgdb (str) – Output gdb that will be zipped and shipped

Returns

Shapefile containing model results at the county level

Return type

str
